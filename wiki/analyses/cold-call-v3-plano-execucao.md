# Cold Call V3 — Plano de Execução (4 Semanas)

> **Para execução:** leia uma tarefa por vez. Não pule. Cada tarefa tem uma verificação ao final — só avance se ela passar.

**Goal:** Sair de poucas calls isoladas para 15 calls/dia consistentes até o fim da semana 4, com script V3 fluindo naturalmente e pelo menos 3 reuniões marcadas.

**Arquitetura:** D0 (setup) → Semana 1 (fluência, 3 calls/dia) → Semana 2 (gatekeeper, 5/dia) → Semana 3 (reuniões, 8/dia) → Semana 4 (volume, 15/dia). A rampa existe para matar call reluctance com volume progressivo — não com força de vontade. O nervosismo pré-discagem some em torno da call 30–40 acumulada.

**Base:** [[estrutura-cold-call-v3]], [[concepts/habitos-atomicos]], [[concepts/negociacao-voss]], [[concepts/trabalho-profundo]], [[plano-30-dias-avalyse-encerrado]]

---

## Tarefa 0 — Setup (fazer ANTES da primeira call)

**Arquivos/materiais:**
- Criar: planilha de tracking (Google Sheets)
- Imprimir ou deixar aberto na tela: script V3 completo
- Abrir: lista de leads com nome do dono já consolidado

---

- [ ] **Passo 1: Criar a planilha de tracking**

Abra o Google Sheets. Crie uma aba chamada `Calls`. Crie estas colunas na linha 1:

```
Data | Lead | Nicho | Nome Dono | Tel | Nome correto? | Gatekeeper passou? | Dono atingido? | Resposta dono | Reunião marcada? | Notas
```

Salve como `Avalyse — Tracking Cold Calls`.

- [ ] **Passo 2: Imprimir ou deixar o script V3 visível na tela**

Abra [[estrutura-cold-call-v3]] no Obsidian em janela separada. Deixe visível durante todas as calls. Nas primeiras 2 semanas você VAI precisar olhar — isso é normal e não atrapalha a call.

- [ ] **Passo 3: Definir o ritual de abertura do bloco**

Anote numa nota adesiva ou no topo da planilha a seguinte sequência (leia antes de cada bloco para o resto da vida):

```
1. Headphones
2. Abrir planilha
3. Abrir script
4. Primeiro número — sem pausa
```

Aplicação direta da Regra dos 2 Minutos de [[concepts/habitos-atomicos]]: a decisão de ligar acontece uma vez (quando você senta). Dentro do bloco, você executa — não decide.

- [ ] **Passo 4: Definir o anchor habit (habit stacking)**

Escreva numa nota:

```
"Depois de chegar da escola e ligar o computador, abro a planilha de calls e discoo o primeiro número."
```

Isso ancora o bloco num hábito que já existe (ligar o computador ao chegar em casa). Fonte: [[concepts/habitos-atomicos]].

- [ ] **Passo 5: Verificar a lista**

Abra sua lista de leads. Confirme que cada entrada tem: nome do negócio, telefone, nicho, posição no Maps, nome do dono (do CNPJ). Se algum lead estiver sem o nome do dono, marque como "sem nome" — você usará o script fallback de V2 nesses casos.

- [ ] **Verificação D0**

Antes de fechar o dia: planilha criada? Script aberto? Ritual de 4 passos escrito? Habit stacking definido? Lista revisada?

Se sim → Tarefa 0 concluída. Amanhã começa a Semana 1.

---

## Tarefa 1 — Semana 1: Fluência (3 calls/dia)

**Objetivo da semana:** escutar sua própria voz no script V3. Não é reunião — é fluência. Errar aqui é o plano.

**Bloco fixo:** 16h30–17h30 (após escola). Primeira call em até 60 segundos de sentar.

**Volume:** 3 calls/dia útil × 5 dias = 15 calls na semana.

---

- [ ] **Passo 1: Bloco D1 — primeiras 3 calls**

Sente, execute o ritual de 4 passos. Abra a planilha. Disca o primeiro número da lista.

Script FASE 1 com nome:
> "Oi, tudo bem? Aqui é o Thiago. O [Nome do Dono] está disponível?"

Independente do resultado (passou, ocupado, nome errado), preencha a linha na planilha imediatamente após desligar. Não espere o final do bloco.

Faça 3 calls. Feche o bloco.

- [ ] **Passo 2: Revisão pós-bloco (5 min)**

Olhe as 3 linhas da planilha. Responda mentalmente:
- Qual parte do script travou mais?
- O nome do dono funcionou na abertura?
- Em que momento perdeu o fio?

Anote 1 coisa na coluna "Notas" da última linha do dia.

- [ ] **Passo 3: Repita D2–D5**

Mesmo ritual. Mesmo bloco. 3 calls/dia.

A cada dia que o nervosismo pré-discagem aparecer, lembre: é antecipação, não perigo. Some depois da primeira call.

- [ ] **Passo 4: Review da semana (domingo, 20 min)**

Abra a planilha. Calcule:
- Quantas calls completadas: ___ / 15
- Quantas vezes o nome do dono funcionou direto: ___
- Quantas vezes o nome estava errado (~10%): ___
- Quantas vezes chegou ao FASE 2: ___

- [ ] **Verificação Semana 1**

≥10 calls completadas (de 15)? Planilha preenchida todos os dias? Script V3 FASE 1 já soa mais natural que no D1?

Se sim → avança para Semana 2.
Se não → repete mais 1 semana no volume de 3 calls/dia antes de escalar.

---

## Tarefa 2 — Semana 2: Gatekeeper (5 calls/dia)

**Objetivo da semana:** FASE 1 fluindo sem travar. Chegando consistentemente no dono.

**Volume:** 5 calls/dia × 5 dias = 25 calls na semana (40 acumuladas).

---

- [ ] **Passo 1: Preparar as 4 ramificações do FASE 1 antes do bloco**

Releia as 4 situações do FASE 1 no script V3:

| Situação | Sua resposta |
|---|---|
| Passa direto | → FASE 2 |
| "É sobre o quê?" | "Rodei uma análise do perfil de vocês no Google Maps — queria passar direto pra ele." |
| "Tá ocupado" | "Sem problema. Quando seria um bom horário para ligar de volta?" |
| Nome errado | "Me desculpe, pode ser que minha informação esteja desatualizada — quem seria o responsável?" |

Leia em voz alta 2 vezes antes de discar. Isso é o "aquecimento" do bloco.

- [ ] **Passo 2: Bloco diário — 5 calls**

Mesmo ritual de 4 passos. 5 calls em sequência. Preencha a planilha após cada call.

Nas calls em que chegar ao FASE 2, use o diagnóstico completo:
> "[Nome], hoje quando alguém busca '[nicho] em [bairro]' no Maps, o [negócio] aparece em média na [X]ª posição. Os 3 primeiros concentram quase 75% dos cliques..."

Se travar no FASE 2, não tem problema. Anote onde travou. Isso é dado.

- [ ] **Passo 3: Praticar objeções (15 min, 3× na semana)**

Na segunda, quarta e sexta, após o bloco de calls, faça 15 min de role-play solo em voz alta para as 4 objeções mais comuns:

**"Manda por WhatsApp":**
> "Claro, posso mandar. Mas seria muito mais rico te mostrar ao vivo — são 10 minutos. Você teria [dia] às [hora]?"

**"Não tenho interesse":**
> "Entendo [Nome]. Posso te perguntar — o que te fez dizer isso? É o timing, ou tem algo específico?" *(Calibrated question — Voss)*

**"Vou pensar":**
> "Faz sentido. Se eu reservar [dia] às [hora] por 10 minutos, você consegue dar uma olhada antes e já vir com as dúvidas?"

**"Quanto custa?":**
> "Antes de falar de preço, quero te mostrar o que a ferramenta faz pelo seu negócio especificamente. São 10 minutos. Você teria [dia]?"

- [ ] **Passo 4: Review da semana (domingo, 20 min)**

Calcule na planilha:
- Calls completadas: ___ / 25
- % calls que chegaram ao FASE 2: ___
- % gatekeeper passou com nome: ___
- % nome estava errado: ___
- Reuniões marcadas: ___

- [ ] **Verificação Semana 2**

≥20 calls completadas (de 25)? FASE 1 com nome funcionando consistentemente? Chegando ao dono em ≥30% das tentativas?

Se sim → avança para Semana 3.
Se não → repete mais 1 semana em 5 calls/dia antes de escalar.

---

## Tarefa 3 — Semana 3: Primeiras Reuniões (8 calls/dia)

**Objetivo da semana:** marcar ≥2 reuniões no Zoom. O FASE 2 já tem volume suficiente para isso acontecer.

**Volume:** 8 calls/dia × 5 dias = 40 calls na semana (≥80 acumuladas — zona em que call reluctance some).

---

- [ ] **Passo 1: Adicionar coluna à planilha**

Adicione uma coluna: `Demo agendada para (data/hora)`. Quando uma reunião for marcada, preencha imediatamente.

- [ ] **Passo 2: Bloco diário — 8 calls**

Mesmo ritual. 8 calls em sequência. Preencha a planilha após cada call.

A partir desta semana o CTA binário é obrigatório em todas as calls que chegarem ao dono:
> "Você consegue **[dia próximo] às [hora]** ou **[dia seguinte] às [outra hora]**?"

Nunca pergunta aberta. Sempre duas opções concretas.

- [ ] **Passo 3: Quando marcar uma reunião**

Imediatamente após desligar:
1. Preenche a planilha com data/hora da reunião
2. Envia um WhatsApp de confirmação:
   > "[Nome], confirmando nossa conversa rápida de 10 minutos no Zoom — [dia] às [hora]. Te mando o link aqui embaixo."
3. Gera o link do Zoom e envia.

- [ ] **Passo 4: Pós-reunião (se acontecer nesta semana)**

Após cada demo realizada, preencha na planilha: `Iniciou trial? (S/N)` e `Notas da demo`. Revise o [[sources/avalyse-demo-meeting-script-v1]] antes de cada demo agendada.

- [ ] **Passo 5: Review da semana (domingo, 25 min)**

Calcule:
- Calls completadas: ___ / 40
- Reuniões marcadas: ___ (meta: ≥2)
- Reuniões realizadas: ___
- Trials iniciados: ___
- Taxa calls → reunião: ___%

- [ ] **Verificação Semana 3**

≥30 calls completadas (de 40)? ≥1 reunião marcada? Script V3 completo (FASE 1 + FASE 2 + CTA) rodando sem consultar o script?

Se sim → avança para Semana 4.
Se não → repete mais 1 semana em 8 calls/dia.

---

## Tarefa 4 — Semana 4: Volume (15 calls/dia)

**Objetivo da semana:** funil previsível. 75 calls na semana, métricas estáveis o suficiente para projetar quantas calls = 1 reunião.

**Volume:** 15 calls/dia × 5 dias = 75 calls na semana (≥155 acumuladas total).

---

- [ ] **Passo 1: Ampliar a janela do bloco**

Com 15 calls, o bloco de 16h30–17h30 pode não ser suficiente. Estenda para 16h30–18h00. Se não der, use também a janela de reposição 20h–20h30 para completar o volume.

- [ ] **Passo 2: Bloco diário — 15 calls**

Mesmo ritual. Sem negociação de volume.

A cada 5 calls, faça uma pausa de 2 minutos. Releia rapidamente o mapa de objeções do script V3 antes de continuar.

- [ ] **Passo 3: Calcular a equação do funil**

No meio da semana (quarta-feira), abra a planilha e calcule:

```
Calls totais acumuladas
÷ Reuniões marcadas acumuladas
= Calls por reunião
```

Exemplo: 60 calls → 3 reuniões = 1 reunião a cada 20 calls.

Esse número é o ativo mais valioso que você constrói nessa semana. Com ele, você sabe exatamente quantas calls precisa por dia para ter X reuniões por semana.

- [ ] **Passo 4: Identificar o gargalo atual**

Use a tabela de diagnóstico abaixo. Olhe sua planilha e determine onde está a maior perda:

| Sintoma | Diagnóstico | Ação |
|---|---|---|
| Gatekeeper não passa (<30%) | Nome errado frequente ou abordagem soa vendedor | Revisar abertura; testar variação de tom |
| Dono desliga antes do diagnóstico | Abertura longa demais | Encurtar para: posição + dado dos 75% + pergunta de processo |
| Chega ao CTA mas não marca | Oferta fraca ou horário ruim | Testar "amanhã cedo" e "após às 18h"; reforçar trial grátis |
| Marca reunião mas não aparece | Confirmação fraca | Enviar WhatsApp de confirmação + lembrete 1h antes |

- [ ] **Passo 5: Review final da Semana 4 (domingo, 30 min)**

Calcule métricas finais das 4 semanas:

| Métrica | Semana 1 | Semana 2 | Semana 3 | Semana 4 |
|---|---|---|---|---|
| Calls feitas | | | | |
| Gatekeeper passou | | | | |
| Dono atingido | | | | |
| Reuniões marcadas | | | | |
| Trials iniciados | | | | |

Calcule taxa calls → reunião. Isso define o volume de calls necessário para bater sua meta de demos por semana.

- [ ] **Verificação Semana 4**

≥60 calls completadas (de 75)? Taxa calls → reunião calculada? ≥3 reuniões marcadas nas 4 semanas? Gargalo principal identificado?

Se sim → sistema instalado. Próximo passo: aumentar volume ou otimizar o gargalo identificado, com base nos dados.

## Regras inquebrável do sistema

1. **Planilha preenchida toda noite.** Mesmo no dia que rendeu zero. Sem dado, não há aprendizado. Aplicação de [[concepts/construir-medir-aprender]].

2. **Primeira call em até 60 segundos de sentar.** O nervosismo vive na antecipação, não na call. Discar é o antídoto.

3. **Se perdeu o bloco das 16h30 → janela de reposição às 20h.** Se perdeu as duas → dia conta como zero, sem compensação amanhã.

4. **Volume antes de perfeição.** Nas semanas 1 e 2, o objetivo não é reunião — é fluência. Uma call imperfeita completada vale mais que um script perfeito não usado.

## Identidade (aplicando [[concepts/habitos-de-identidade]])

Não é "preciso fazer 3 calls hoje". É:

> **"Sou um fundador que conversa com prospects todo dia, mede tudo, e ajusta rápido."**

Cada call é um voto para essa identidade. Calls ruins também contam — você está votando que é alguém que faz a coisa mesmo quando é difícil. Fonte: [[concepts/habitos-atomicos]], [[concepts/habitos-de-identidade]].
